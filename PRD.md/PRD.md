## 제품 요구사항 문서 (PRD) - Face Calendar Landing

### 문서 버전
- 버전: 0.1
- 작성일: 2025-11-11

## 1. 제품 개요
- **목표**: 랜딩 페이지를 통해 핵심 가치 제안 전달과 함께 리드(이름/이메일/전화번호) 수집, 체험 진입 유도.
- **핵심 메시지**: 사진 한 장으로 시작하는 개인 맞춤 스킨케어 루틴.
- **레퍼런스 파일**: `app/page.tsx` (구성은 단일 파일에서 필요한 섹션 컴포넌트를 불러와 사용).

## 2. 대상 사용자
- **주요 대상**: 스킨케어에 관심이 있으나 제품 선택이 어려운 일반 사용자.
- **환경**: 모바일/데스크톱 웹 브라우저(모바일 우선), 카메라 사용 가능 단말 고려.

## 3. 핵심 기능 (Landing)
- **네비게이션**: 상단 고정 내비게이션(`components/navigation.tsx`).
- **히어로 섹션**: 가치 제안, 메인이미지, CTA 버튼 2종.
  - CTA1: 모달 열림 → 리드 수집 폼 제출 (`actions/supabase.ts` 서버 액션 `submitLead`).
  - CTA2: 진단/체험 페이지 경로 진입(`/diagnosis`).
- **문제/해결/혜택/가격/FAQ/최종 CTA**: 신뢰와 전환을 위한 정보 구조.
- **작동 방법(How it works)**: 3단계 프로세스 시각화 + 인터랙션 모달
  - 사진 촬영/업로드 선택 → 카메라 모달(촬영) 또는 업로드 모달(파일 선택) → 미리보기 → 분석 안내 모달.
  - 카메라 사용 시 `getUserMedia`로 스트림 획득, `canvas.toBlob`으로 캡처.

## 4. 사용자 플로우
1) 랜딩 접속 → 히어로 영역 노출
2) CTA1 클릭 → 리드 수집 모달 → 이름/이메일/전화 입력 → 제출 → 성공 토스트/모달 종료
3) 또는 CTA2 클릭 → `/diagnosis` 등 체험 진입
4) “작동 방법” 영역에서 “사진 촬영” 클릭 → 입력 방식 선택(카메라/업로드)
5) 카메라: 권한 허용 → 실시간 프리뷰 → 촬영 → 미리보기 → “분석 시작” 안내 모달
6) 업로드: 파일 선택 → 미리보기 → “분석 시작” 안내 모달

## 5. 화면/섹션 구조
- `app/page.tsx`에서 불러오는 섹션 컴포넌트
  - `Navigation` → `Hero` → `Problem` → `Solution` → `HowItWorks` → `Benefits` → `Pricing` → `FAQ` → `FinalCTA`
- 주요 컴포넌트 포인트
  - `Hero`: 메인이미지 `public/modern-skincare-app-interface-showing-ai-skin-anal.jpg` 사용, CTA 모달에서 리드 수집.
  - `HowItWorks`: 카메라/업로드/분석 모달 상태 관리, 이미지 3종 노출.

## 6. 데이터/폼 요구사항
- 리드 수집 폼 필드: `name`, `email`, `phone` (모두 필수)
- 제출 처리: Next.js 서버 액션(`actions/supabase.ts`)을 통한 저장
- 성공/실패 피드백: 모달 닫기 + 알림(토스트/문구)

## 7. 에셋/이미지
- `public/modern-skincare-app-interface-showing-ai-skin-anal.jpg` (히어로)
- `public/step-1-photo.png` (작동 1단계)
- `public/step-2-ai-analysis.png` (작동 2단계)
- `public/step-3-recommendation.png` (작동 3단계)
- 플레이스홀더: `public/placeholder.jpg`

## 8. 로깅/계측(필수)
- **클라이언트 로그(개발용)**: 핵심 인터랙션에 `console.log` 추가
  - 히어로 CTA 클릭, 리드 제출 성공/실패, 카메라 권한 요청/획득/오류, 사진 캡처 성공/실패, 업로드 선택/미리보기, “분석 시작” 클릭.
- **서버 로그**: 서버 액션(`submitLead`)에서 요청 수신/성공/실패 로그 남김.
- 추후 분석 도구 연동(GA/Amplitude 등) 시 동일 이벤트 키로 매핑.

## 9. 비기능 요구사항/제약
- **스타일**: Tailwind CSS만 사용, 간단 클래스 조합. 불필요한 컴포넌트 분리 지양.
- **API/아키텍처**: 모든 API 호출은 Next.js 서버 액션 사용. 노션 API는 `lib/notion.ts`, `actions/notion.ts`에만.
- **상태관리**: 최소화. 복잡한 리듀서 지양.
- **패키지 매니저**: pnpm.
- **이미지 처리**: `next.config.mjs`에서 `images.unoptimized=true` 설정 반영됨.

## 10. 접근성/UX
- 모달: 키보드 포커스 트랩, `Dialog` 컴포넌트의 접근성 속성 준수.
- 버튼/아이콘: 적절한 `aria-label`, 텍스트 대비 확보.
- 반응형: 모바일 우선, grid/gap로 레이아웃 유연성 유지.

## 11. 성능/보안
- 이미지: 정적 에셋 활용, 필요 시 사이즈 최적화(가로/세로 비율 유지).
- 카메라 권한: 사용자 명시 동의 필요, 권한 거부 시 우회 경로(업로드) 안내.
- 개인정보: 리드 데이터 최소 수집, 서버 액션에서 예외 처리 및 로깅.

## 12. 성공 지표(KPI)
- 히어로 CTA 클릭률
- 리드 폼 제출 전환율
- HowItWorks 모달 오픈률/완료율(촬영/업로드/분석 시작)
- 이미지 로드 실패율(플레이스홀더 대체 비율)
